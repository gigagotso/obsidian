# Shortcut Tags Are Not Applied to Conversations

## Issue

Shortcuts can have tags assigned to them. When a shortcut is used and the message is sent from the composer, the shortcut's tags should automatically be attached to the conversation.

Currently, the message is sent successfully, but the tags are not applied to the conversation.

## Expected Behavior

- A shortcut has one or more tags assigned.
- An agent uses the shortcut and sends the message.
- All tags assigned to the shortcut are automatically attached to the conversation.
- Existing conversation tags are preserved.
- Duplicate tags are not created.
- The UI updates to display the newly attached tags.
## Goal

Ensure that whenever a shortcut is used to send a message, all tags assigned to that shortcut are automatically attached to the conversation.
