# DateTime Component Calendar Issues

The DateTime component's calendar behavior is currently confusing and does not follow standard date picker conventions.

For example, when selecting an End Date, the Start Date is already pre-selected or highlighted, resulting in unexpected behavior and a poor user experience.

## Tasks

- Investigate and fix the date selection logic.
- Ensure the calendar behaves like a standard date picker/range picker.
- Verify that Start Date and End Date selection states are handled correctly.
- Remove any unnecessary, duplicated, legacy, or dead code related to date selection and calendar state management.
- Simplify the implementation where possible and improve maintainability.

## Expected Result

The DateTime component should behave predictably and consistently, following common calendar UX patterns without unexpected pre-selections or state conflicts.
